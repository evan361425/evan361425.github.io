    Vault 實作¶

    Server 設定¶

    {
    "storage": {
        "mysql": {
            "address": "127.0.0.1:3306",
            "database": "playground",
            "table": "vault",
            "username": "root",
            "password": "mysql-password",
            "plaintext_connection_allowed": true,
            "ha_enabled": true
        }
    },
    "listener": {
        "tcp": {
            "tls_disable": true
        }
    },
    "seal": {
        "awskms": {
            "region": "ap-northeast-1",
            "access_key": "aws-iam-access-key",
            "secret_key": "aws-iam-secret-key",
            "kms_key_id": "aws-kms-key-id"
        }
    },
    "ui": true,
    "disable_mlock": true,
    "api_addr": "http://127.0.0.1:8200",
    "cluster_addr": "https://127.0.0.1:8201"
}

    $ vault server -config=config.json

    Seal¶

    重新封印並再解封的話要輸入 Recovery Key

    注意：若重新申請一個 Root Token，也會需要 Recovery Key

    $ vault operator seal

    Secrets¶

    key-value 的資料庫

    $ vault secrets enable -path=secret kv
$ vault kv put secret/hello foo=world
$ vault kv list secret
$ vault kv get secret/hello

    Authentication¶

    先允許 GitHub 登入

    $ vault auth enable github
$ vault write auth/github/config organization=104corp

    登出¶

    $ rm ~/.vault-token

    GitHub¶

    存取權杖¶

    去 GitHub 設定頁 > Developer settings > Personal access tokens > Generate new token

    權限要有：admin:org > read:org，讓 Vault 可以讀取你帳號的組織

    Policy¶

    讀取現有的政策¶

    $ vault read sys/policy

Key         Value
---         -----
keys        [default root]
policies    [default root]

    建立政策¶

    $ vault policy write my-policy policy.hcl

    policy.hcl 檔：

    path "secret/*" {
  capabilities = ["create", "read", "update", "delete", "list"]
}

path "secret/hello" {
  capabilities = ["deny"]
}

path "secret/restricted" {
  capabilities = ["create"]
  allowed_parameters = {
    "foo" = []
    "bar" = ["zip", "zap"]
  }
}
